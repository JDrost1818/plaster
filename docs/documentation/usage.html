Usage

Plaster is a command line tool which provides a multitude of options in order to change behavior. The general syntax of the tool is as follows:

The basic syntax can be broken down into four parts:

  • Mode
  • Scope
  • Class Information
  • Configuration

Mode

The mode is the first argument and specifies is to be accomplished. There are three current modes for which Plaster supports.

Generate

The most common mode. Anytime line(s) of code will be added to the project, generate will be used. Whether it is adding a new model, adding a new field to an already existing model, or adding other classes, generate will be the mode to choose.

Delete

This is the exact opposite of generate. It can remove whole swaths of code from the repository or you can pinpoint individual fields.

Modify

This is the most specialized mode. The only thing that can be modified are names/types. Use this when you have existing fields that are either the wrong type or the wrong name.

Scope

The scope defines the extent to which Plaster is going to perform the mode. In order to more easily understand the process, remember which files depend on each other. Without being pedantic, notice how each file relies on all files above it in the hierarchy:

Model
Repository
Service
Controller

Therefore, something below cannot exist without everything above. All those in the hierarchy are available as a scope. When using as a scope, Plaster will not just do the single file, rather, will do all those required. So when generating or deleting, recognize all files the will be affected. Anything above will be generated and anything below will be deleted whenever the respective modes are activated.

Scaffold

Scaffold is a special scope keyword used to indicate all modes should be activated. This is semantically the same as using the controller scope during generation and model during deletion; however, it is included as a more intuitive option.

Note

The modify mode is the only mode that has restricted access to scopes. It can only be used in tandem with the model scope.
